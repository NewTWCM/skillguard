Threat Intelligence

Why SkillGuard?

Real-world incidents that drove our three-layer defense design.

🚨

The gstack Telemetry Backdoor

In early 2026, a popular Claude Code skill named gstack was found silently exfiltrating user insights and business data. SkillGuard detected this bypass using Layer 1 (Honeypot) and Layer 2 (Behavioral Scan).

🛡️

Tool & Environment Hijacking

Advanced attackers attempt to override system binaries like shasum or grep. SkillGuard v2.3 uses Absolute Path Locking and $PATH sanity checks to ensure security tools remain untampered.

Quick Start

Get Protected in 3 Steps

1

Install & Seal

Deploy honeypots and compute the Layer 3 checksum to prevent tampering.

terminal
$ chmod +x skillguard.sh
$ ./skillguard.sh install
[L3] v2.3 Multi-Link Seal Applied.
2

Full Security Scan

Run on any plugin or skill directory before use.

terminal
$ ./skillguard.sh scan /path/to/suspicious-skill
V11 (Time-Bombs): 0 hits
V12 (Env-Hijack): 0 hits
V15 (OOB Inject): 0 hits (CRITICAL)
[✔] v2.3 Scan complete. Security posture: EXTREME.
3

Routine Check

Check honeypot integrity and self-integrity regularly.

terminal
$ ./skillguard.sh check
System and PATH Integrity: SECURE.

Research & Citation

Abstract

The rapid evolution of LLM-based agentic ecosystems has introduced a critical attack surface: third-party plugins that execute with elevated privileges. We present SkillGuard, a lightweight, three-layer runtime defense system. Our evaluation demonstrates high detection rates with minimal performance overhead (<1s).
